Most solutions for programming and maths puzzles you'll find online are
presented as a "finished product": the code you'd need to write to solve the
problem, as well as maybe the theory behind it.
Write-ups like that are often fascinating, and quite useful, but it's often not
clear *how* you'd get to the solution.
Frequently some intermediate step will require insight: "here, we notice xyz,
and can see that the problem is a special case of this other one".

Today, I want to write about a particular small problem I solved with almost no
insight at all.

every step was just the application of some trick or technique
that I've used before.

, and I managed to get a working solution without
understanding *any* of the theory!

# The Problem

The problem was concerned with a sequence made from pairs of numbers, starting
with two numbers, $n$ and $m$.
If the numbers are equal, the sequence stops.
Otherwise, the smaller number is subtracted from the larger, and the smaller
number is doubled.
The objective was to find, given $n$ and $m$, whether the sequence eventually
stopped, or went into a loop.
Here are some examples:

```
     3, 5
     6, 2
     4, 4
---- done ----
```

```
     1, 4
     2, 3
     1, 4
---- done ---- 
```

# Step 1: Write a Dumb Solution

```python
def sequence(n,m):
    while n != m:
        yield (n,m)
        if n < m:
            m -= n
            n *= 2
        else:
            n -= m
            m *= 2

def loops(n,m):
    seen = set()
    for x in sequence(n,m):
        if x in seen:
            return True
        else:
            seen.add(x)
    return False
```
